# Multimodal Sports Injury Risk Prediction

This repository contains code and results for a multimodal athlete injury-risk prediction system. It includes time-series deep models (GRU, LSTM, Transformer) and classical tree-based models (Random Forest, XGBoost, LightGBM) together with evaluation, calibration and explainability (SHAP) tools.

---

## Project overview
- Goal: predict athlete injury risk (Healthy / Low Risk / High Risk) from multimodal features (physiological, performance, contextual).
- Models included: GRU, LSTM, Transformer, RandomForest, XGBoost, LightGBM.
- Components: data preprocessing (sliding windows), model training, calibration, evaluation, SHAP explainability and visualization of temporal trajectories.

## Dataset
- Main dataset: `multimodal_sports_injury_dataset.csv` (root).
- A synthetic unseen test set is included for Transformer tests: `Transformer/Test_transformers_unseen_data/synthetic_unseen_test_dataset.csv`.

## Repository structure (key files)
- `run.ipynb` — exploratory / demo notebook.
- `GRU/` — GRU training/eval: `main_gru.py`, `model.py`, `calibrate_gru.py`, `generate_plots.py`. Trained model & results in `GRU/GRU_results/`.
- `LSTM/` — LSTM training/eval: `main_lstm.py`, `lstm.py`, `sliding_windows.py`, `calibrate_lstm.py`, `generate_plots.py`. Trained models & results in `LSTM/results/`.
- `Transformer/` — Transformer training/eval: `main_trans.py`, `model.py`, `calibrate_transformers.py`, plus unseen-data test in `Test_transformers_unseen_data/`.
- `tree_models/` — classical models and SHAP: `RF.py`, `XGB.py`, `LightGBM.py`, `SHAP.py` and results in `tree_models/results/`.

## Requirements
- Python 3.8+ recommended.
- Typical packages used:
  - numpy, pandas, scikit-learn
  - matplotlib, seaborn
  - torch (for GRU/LSTM/Transformer implementations)
  - xgboost, lightgbm
  - shap

Create a virtual environment and install dependencies (example):

```bash
python -m venv venv
venv\Scripts\activate    # Windows
pip install -U pip
pip install numpy pandas scikit-learn matplotlib seaborn torch xgboost lightgbm shap
```

If you prefer, create a `requirements.txt` with the packages above.

## Quick start
1. Inspect data and experiments: open `run.ipynb`.
2. Train / evaluate LSTM:

```bash
# from project root
python LSTM/main_lstm.py
```

3. Train / evaluate GRU:

```bash
python GRU/main_gru.py
```

4. Train / evaluate Transformer:

```bash
python Transformer/main_trans.py
```

5. Run tree-based experiments (example Random Forest):

```bash
python tree_models/RF.py
```

Notes: each `main_*.py` has in-script options for dataset paths, hyperparams, training vs. evaluation modes and saving outputs. Use `calibrate_*.py` scripts to compute/plot calibration curves and apply sigmoid calibration.

## Evaluation & explainability
- Confusion matrices, calibration curves and temporal risk trajectories are generated by the `generate_plots.py` scripts located in the model folders.
- SHAP summary and dependence plots are produced by `tree_models/SHAP.py` and by model-specific `generate_plots.py` scripts.

## Saved models & results
- GRU: `GRU/GRU_results/` (best models, grid-search and calibration results).
- LSTM: `LSTM/results/` (best models, grid-search, calibration results).
- Transformer: `Transformer/Transformers_results/` and `Transformer/Test_transformers_unseen_data/results/`.
- Tree models: `tree_models/results/` with per-algorithm subfolders.

## Reproducing figures
- Each model folder contains a `generate_plots.py` script to reproduce plots shown in experiments (confusion matrices, calibration curves, SHAP plots, temporal trajectories). Example:

```bash
python GRU/generate_plots.py
python LSTM/generate_plots.py
python Transformer/generate_plots.py
python tree_models/SHAP.py
```

## Running inference on unseen data (Transformer)
- Example test script: `Transformer/Test_transformers_unseen_data/main_test_unseen.py`.
- Replace the CSV with your own unseen dataset or use `synthetic_unseen_test_dataset.csv` as a template.

## Notes & recommendations
- Ensure consistent preprocessing (feature scaling, windowing) when switching between training and inference.
- For reproducibility, fix random seeds in each `main_*.py` and document used hyperparameters (grid search results are stored in each results folder).
